Detailed Description

Interface for querying locked chain state, used by legacy code that assumes state won't change between calls.

New code should avoid using the Lock interface and instead call higher-level Chain methods that return more information so the chain doesn't need to stay locked between calls.

Return height of the first block in the chain with timestamp equal or greater than the given time and height equal or greater than the given height, or nullopt if there is no such block.

Calling this with height 0 is equivalent to calling findFirstBlockWithTime, but less efficient because it requires a linear instead of a binary search.
